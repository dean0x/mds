**Potential napi build conflict with hand-written index.js** - `crates/mds-napi/index.js` / `.gitignore:5`
**Confidence**: 80%
- Problem: The `.gitignore` previously ignored `crates/mds-napi/index.js` because `@napi-rs/cli` generates that file when running `napi build --platform`. The PR removes it from `.gitignore` and commits a hand-written multi-platform loader. If a future contributor (or CI) runs `napi build --platform` without `--no-js`, it will silently overwrite the hand-written `index.js` with napi's auto-generated version, breaking the custom loader logic.
- Fix: Add a comment at the top of `crates/mds-napi/index.js` warning that it is hand-maintained and not auto-generated. Additionally, ensure CI or build scripts use `--no-js` when invoking `napi build --platform`. Example header:
  ```js
  // HAND-MAINTAINED: This loader is NOT auto-generated by @napi-rs/cli.
  // Do not run `napi build --platform` without --no-js, or this file will be overwritten.
  ```
